ECMAScript 5 has a mode you has a developer can enabled to restrict aspects of JavaScript called Strict Mode. It can be enabled in an entire script or in functions by using the "use strict"; before any statements. But what exactly does it do?

Strict mode will force users to declare variables to prevent pollution of global scope.

  //Before
 v = "This is not strict mode";
 //Runs

 //After Error
 "use strict";
 v = "This is strict mode";
 //Throws Error

 //After Correct
 "use strict";
 var v = "This is strict mode";
 //Runs

It will also not allow a property to be declared more than once.

  //Will run
var obj = {test: true, test: true};

//Throws Error
"use strict";
var obj = {test: true, test: true};

Any attempt to use keyword eval will result in an error.

//All Errors
"user strict";

var obj = {};
obj.eval = "Anything";

function test(eval) {
   //Anything
}

var test = function(eval) {
   //Anything
}

Will not allow repeat names of arguments in a function.

"use strict";
//Throws Error
function test (a,a) {
  //Anything
}
